# 프로젝트 코딩 규칙 (KDT 실습)

## 좋은 코딩의 방향성
- 복잡한 코드가 아닌, 단순하고 흐름을 파악할 수 있는 코드를 지향한다.
- 카멜기법 혹은 스네이크 케이스 등 프로젝트 내 개발 표준을 일관되게 준수한다.
- 적절한 주석으로 제3자가 보더라도 유지보수가 가능하도록 한다.

## 파일 상단 표준 머리말(필수)
모든 `.py` 파일 상단에는 아래 형식의 머리말을 유지한다.

```python
# ----------------------------------------------------------------------------
# 작성자   : 광주캠퍼스_2반_정구현
# 작성목적 : (이 파일이 하는 일 요약)
# 작성일   : YYYY-MM-DD
#
# 본 파일은 KDT 교육을 위한 Sample 코드이므로 작성자에게 모든 저작권이 있습니다.
#
# 변경사항 내역 (날짜, 변경목적, 변경내용 순으로 기입)
#   YYYY-MM-DD | 변경목적 | 변경내용
# ----------------------------------------------------------------------------
```

## 변경 시 규칙 (중요)
- **파일을 수정할 때마다** 머리말의 `변경사항 내역`에 한 줄을 추가한다.
  형식: `YYYY-MM-DD | 변경목적 | 변경내용`
- 최신 변경이 아래쪽에 오도록 시간순으로 누적한다(기존 이력은 삭제하지 않는다).
- 날짜는 실제 수정한 날짜를 사용한다.
